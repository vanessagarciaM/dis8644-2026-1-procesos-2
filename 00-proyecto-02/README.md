# proyecto-02

## estructura del proyecto

cada grupo de trabajo se hace cargo de un tema específico.

cada grupo entrega 2 alternativas de diseño, con sus respectivos esquemáticos, PCB, lista de materiales, entre otros.

## convenciones de archivos

### README.md

cada grupo debe entregar un README.md con la estructura del proyecto propuesta en la plantilla subida.

### videos

no subir videos al repositorio, subirlos a Youtube o similar, enlazarlos.

### imágenes

subir imágenes al repositorio a una subcarpeta imágenes, enlazarlas en README.md.

será penalizado no subir las imágenes de forma correcta, con -2 puntos, o sea nota máxima 5.0. por ejemplo, NO subir imágenes con método "drag-and-drop" directo a README.md y que GitHub haga el enlace por ustedes.

### proyectos de KiCad

Cada grupo debe subir los proyectos de KiCad como carpeta no comprimida.

Queremos que los nombres de los archivos sean claros, por eso tendrán esta estructura:

las carpetas tendrán tres palabras clave, separadas por guiones entre ellas:

NOMBRE-ALTERNATIVA-VERSION/

los valores para cada variable son:

NOMBRE - en orden de grupos, los valores posibles son

- piezo
- secuenciador
- oscilador1
- oscilador2
- filtro
- percusion

ALTERNATIVA - valores posibles son  "01" o "02".

VERSION - valores posibles son desde "v01", "v02" hasta la última que hagan. esto permite entender el flujo de trabajo y la evaluación del proyecto.

Con esto, dentro de cada carpeta, los archivos tendrán estos mismos nombres, pero con extensiones .sch, .pcb, .kicad_pro, de hecho solamente queremos esos 3.
